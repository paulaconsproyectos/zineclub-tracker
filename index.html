Infraestructura web
Dia 1
Comprar dominio zineclub.io en Namecheap
P Pa
namecheap.com > buscar 'zineclub.io' > añadir al carrito > pagar con tarjeta. Activa Domain Privacy gratis. ~30€/año. Sin dominio propio pareces hobby project, no producto vendible.
NAMECHEAP ~30€
Dia 1
Apuntar DNS de zineclub.io a Vercel
P Pa
Namecheap > Domain List > Manage > Advanced DNS. Registro A: @ > 76.76.21.21. CNAME: www > cname.vercel-dns.com. Tarda 10-30 min en propagar. Hasta que no propague, la landing no carga.
DNS NAMECHEAP
Dia 1
Añadir zineclub.io como dominio en Vercel
P Pa
Vercel dashboard > tu proyecto > Settings > Domains > Add > 'zineclub.io'. Vercel verifica el DNS automaticamente. La landing ya carga con HTTPS desde el dominio propio.
VERCEL gratis
Dia 1
Configurar subdominio app.zineclub.io a GitHub Pages
P Pa
En Namecheap: CNAME: app > paulaconsproyectos.github.io. En repo 'zineclub' de GitHub > Settings > Pages > Custom domain: app.zineclub.io. Crear fichero CNAME en la raiz del repo con el texto 'app.zineclub.io'.
GH PAGES gratis
Dia 2
Verificar que HTTPS funciona en ambos dominios
P Pa
Vercel y GitHub Pages generan el certificado SSL automaticamente. Abrir https://zineclub.io y https://app.zineclub.io en el movil. Si sale el candado verde, listo. Si no, esperar 20 min mas y recargar.
SSL AUTO gratis
Dia 2
Analizar velocidad con PageSpeed Insights y anotar los 3 errores principales
P Pa
pagespeed.web.dev > pegar https://zineclub.io > analizar. Objetivo: puntuacion >80 en movil. Los fallos mas comunes son imagenes sin comprimir y JS bloqueante. Apuntar los 3 mas urgentes y corregirlos.
PAGESPEED gratis
Dia 2
Comprimir todas las imagenes con Squoosh (landing y app)
P Pa
squoosh.app > subir cada imagen > formato WebP > calidad 80% > descargar. Una imagen de 2MB que pesa 150KB hace la app 10x mas rapida. Sustituir en el repo y hacer commit.
SQUOOSH gratis

Añadir meta tags SEO y Open Graph en la landing
P Pa
En el head del HTML: title (max 60 chars), meta description (max 155 chars), og:title, og:description, og:image (1200x630px), og:url. Sin OG tags, cuando alguien comparte en WhatsApp no aparece preview.
SEO META TAGS

Instalar favicon y Apple Touch Icon en landing y app

favicon.ico (32x32) + apple-touch-icon.png (180x180) en la raiz. Sin esto la pestana del navegador sale en blanco y la PWA no tiene icono al guardar en el movil.

Supabase y base de datos
Dia 2
Revisar estructura de la tabla users en Supabase
P Pa
Supabase > Table Editor > users. Verificar columnas: id, email, is_premium, premium_until, stripe_customer_id, created_at, invite_code, invited_by. Sin estas columnas el webhook de Stripe no puede activar Premium.
SUPABASE DB
Dia 2
Activar Row Level Security (RLS) en todas las tablas
P Pa
Supabase > Authentication > Policies. Cada usuario solo lee sus propios datos. Sin RLS cualquier usuario puede leer los datos de todos los demas. Agujero de seguridad critico que invalida la venta.
SUPABASE RLS
Dia 3
Verificar que los backups automaticos de Supabase estan activos
P Pa
Supabase > Settings > Database > Backups. El plan gratuito hace backups diarios 7 dias. Verificar que estan activos. Sin backup, si la DB se corrompe pierdes todos los usuarios y el historico.
BACKUPS AUTO
Dia 3
Crear Edge Function 'stripe-webhook' en Supabase
P Pa
Supabase > Edge Functions > New > 'stripe-webhook'. Codigo: recibe el evento de Stripe, verifica la firma criptografica, actualiza is_premium=true o false segun el tipo de evento. Ver docs en supabase.com.
EDGE FN SUPABASE
